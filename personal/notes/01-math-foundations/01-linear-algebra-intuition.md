## Step 1 — Векторы

**Что такое вектор**
Вектор — это список чисел, который мы рисуем как направленный отрезок.

**Три способа смотреть на вектор**
1) список чисел, определяющий точку в N-мерном пространстве;
2) точка, которая является концом отрезка;
3) стрелка, определяющая его направление.

**Длина вектора**
Определяется по формуле Пифагора и равна квадратному корню из суммы квадратов катетов:
`|v| = sqrt(v1² + v2² + … + vn²)`.

**Сложение векторов**
Для сложения нескольких векторов нужно сложить каждое из их чисел ([2,5] + [−1,3] = [1,8]).
Геометрически — это «достраивание» до параллелограмма (правило треугольника/параллелограмма).

**Умножение на число (скаляр)**
- на число > 1 — длина растёт, направление то же;
- на число в (0, 1) — длина убывает;
- на отрицательное — направление меняется на противоположное (поворот на 180°).

**Нулевой вектор**
Вектор нулевой длины, у которого направление не определено.

> В AI вектором кодируется «всё»: слово → 768 чисел (embedding), картинка → миллионы пикселей, пользователь → вектор предпочтений.

## Step 2 — Матрицы как преобразования

Матрица превращает один вектор в другой: поворачивает, масштабирует, растягивает или проецирует.

Пример — поворот на 90°:
```
M = [[0, -1],
     [1,  0]]
M @ [3, 1] = [-1, 3]
```

**Почему это и есть модель:**
- веса нейросети → матрицы, преобразующие вход в выход;
- attention scores → матрица, решающая «на что смотреть»;
- embeddings → матрица, переводящая слова в векторы.

Применение матрицы к вектору = один слой нейросети в чистом виде.

## Step 3 — Dot product (скалярное произведение) = мера похожести

```
a · b = a1·b1 + a2·b2 + … + an·bn
```

| Знак `a · b` | Геометрия | Смысл |
|---|---|---|
| > 0 | сонаправлены | похожи |
| = 0 | перпендикулярны | не связаны |
| < 0 | противоположны | непохожи |

Это буквально то, как работают поиск, рекомендации и RAG: ищем векторы с бóльшим dot product.
Косинусная похожесть — это dot product, поделённый на длины:
`cos(a,b) = (a·b) / (|a|·|b|)` — нормирует, чтобы не зависеть от длины вектора.

## Step 4 — Линейная независимость и ранг

**Линейная независимость.** Набор векторов независим, если ни один из них нельзя получить
комбинацией остальных. Если можно — набор зависимый, и он «лишний» по информации.

Пример:
```
v1 = [1, 0, 0]
v2 = [0, 1, 0]
v3 = [2, 1, 0]   # v3 = 2·v1 + v2 → зависимый
```
Все три лежат в плоскости xy — добраться до [0,0,1] из них нельзя. Три вектора, но всего
2 степени свободы.

**Для AI:** если в фичах `feature_3 = 2·feature_1 + feature_2`, то такая фича не добавляет
информации и делает веса нестабильными (мультиколлинеарность).

**Базис** — минимальный набор независимых векторов, через которые выражается всё пространство.
Стандартный базис 3D: {[1,0,0], [0,1,0], [0,0,1]}. Но любой набор из 3 независимых векторов —
тоже базис (просто другая система координат).

**Ранг** матрицы = число независимых столбцов = число независимых строк.
- Полный ранг → решение единственное, модель стабильна.
- Ранг меньше → фичи избыточны, решений бесконечно много, нужна регуляризация.
- Ранг 1 → все столбцы — копии одного вектора, данные лежат на прямой.

> LoRA — прямой пример: вместо обновления матрицы весов 4096×4096 (~16M параметров)
> обучают два маленьких: 4096×16 и 16×4096 (~131K). Предположение — обновление живёт
> в 16-мерном подпространстве. Это и есть «линейная алгебра делает реальную работу».

## Step 5 — Проекция

Проекция вектора `a` на направление `b`:
```
proj_b(a) = (a·b / b·b) · b
```
Это «тень» `a` вдоль `b`. Остаток `(a − proj_b(a))` перпендикулярен `b`.

Пример: `a = [3,4]`, `b = [1,0]` → проекция `[3,0]` (отбросили y-компоненту).
Это простейшее снижение размерности: выкинуть направление, которое нам не нужно.

**Где это в ML:**
- линейная регрессия — решение literally проекция наблюдений на пространство столбцов;
- PCA — проектирует данные на направления максимальной дисперсии;
- attention в трансформерах — проекции queries на keys.

## Step 6 — Процесс Грама–Шмидта

Превращает любой набор независимых векторов в **ортонормированный базис**
(все векторы длины 1 и попарно перпендикулярны).

Алгоритм:
1. Первый вектор — нормируем → `u1`.
2. Из следующего вычитаем его проекции на все уже готовые `u` → нормируем.
3. Повторяем для остальных.

```
u1 = v1 / |v1|
w2 = v2 − (v2·u1)·u1;   u2 = w2 / |w2|
w3 = v3 − (v3·u1)·u1 − (v3·u2)·u2;   u3 = w3 / |w3|
```

На этом построена **QR-декомпозиция** (Q — ортонормированный базис, R — коэффициенты проекций).
QR используется в устойчивых численных решателях, для собственных значений и для least-squares
регрессии. Проверка корректности — простая: у всех пар `ui·uj = 0`, а `|ui| = 1`.

## Ментальная карта урока

| Концепт | Что значит по-человечески | Где всплывёт в AI |
|---|---|---|
| Вектор | список чисел = точка/направление | embedding слова/картинки |
| Матрица | преобразование вектора | слой нейросети, веса |
| Dot product | «насколько похожи» | attention, cosine similarity в RAG |
| Лин. независимость | «не дублируют инфу» | отбор фичей, отсутствие мультиколлинеарности |
| Ранг | «сколько реальных степеней свободы» | LoRA, обратимость, обусловленность |
| Проекция | «тень вдоль направления» | регрессия, PCA, attention |
| Грам–Шмидт / QR | «выровнять базис» | устойчивые численные методы |
